# AI RULES – EXECUÇÃO OBRIGATÓRIA

## Regras principais
- Nunca responder sem testar quando aplicável
- Sempre usar Playwright para interação
- Sempre usar DevTools para console e network
- Nunca assumir comportamento sem validação
- Retestar após qualquer correção
- Manter respostas objetivas e técnicas
- Não explicar conceitos básicos se não solicitado
- Se um teste não puder ser executado, parar e explicar o motivo
- Não tentar simular resultados não obtidos

## Confirmação obrigatória
Antes de executar qualquer tarefa:
- Confirme que leu e entendeu este arquivo
- Se não puder seguir alguma regra, explique o motivo

## Relatório obrigatório (curto)
No final da resposta, inclua SOMENTE:
- Playwright: usado / não usado (motivo)
- DevTools: usado / não usado (motivo)
- Viewports testados (se houver)
- Problemas encontrados (resumo)
- Correções aplicadas

Quando a tarefa envolver performance ou métricas:
- Medir desktop e mobile
- Reportar diferença percentual
- Indicar mudança de elemento LCP
